# 抢注功能测试指南

## 流程说明

抢注执行顺序：**飞书「即将抢注」通知** → **WHOIS 查询** → 若可注册则：有 GoDaddy 且任务勾选自动注册则走 GoDaddy 下单，否则更新为「processing」并发飞书「请尽快手动完成注册」。

无 GoDaddy key 时也会跑通：只发通知 + WHOIS + 手动分支（状态改 processing + 飞书结果卡片）。

---

## 方式一：命令行（推荐先测）

无需启动 API、不用 JWT，直接跑一次。

```bash
cd backend

# 按域名测试（会自动创建 pending 任务并执行）
go run ./cmd/snatch_domain -domain=example.com

# 或指定已有任务 ID
go run ./cmd/snatch_domain -task-id=1
```

**预期：**

- 若配置了飞书 Webhook（后台「通知设置」里）：先收到「即将抢注」卡片；若该域名 WHOIS 显示可注册，再收到「域名已可注册，请尽快手动完成注册」。
- 控制台无报错即表示执行完成；可到数据库看 `snatch_tasks` 表该任务状态是否变为 `processing`（可注册时）或保持 `pending`（仍不可注册）。

**配置：** 使用 `backend/etc/domain.yaml`。要发飞书需在库里把「通知设置」的 Webhook 填好（或逻辑里会读 `NotifySettings`）；cmd 会从同一配置文件所在目录解析，并会读 DB 的 `notify_settings` 表拿 Webhook。

## 方式二：API

1. **启动服务**

   ```bash
   cd backend
   go run ./api/domain
   ```

2. **登录拿 Token**

   ```bash
   curl -s -X POST http://localhost:8888/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"你的用户名","password":"你的密码"}' | jq -r '.data.accessToken'
   ```

   把输出的 token 记下来。

3. **触发抢注**

   按域名（无则自动建任务）：

   ```bash
   curl -X POST http://localhost:8888/api/snatch/execute \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer 上一步的token" \
     -d '{"domain":"example.com"}'
   ```

   或按任务 ID：

   ```bash
   curl -X POST http://localhost:8888/api/snatch/execute \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer 上一步的token" \
     -d '{"taskId":1}'
   ```

**预期响应：** `{"code":0,"data":{"msg":"抢注已触发，请关注飞书通知"},"msg":""}`，然后同上：看飞书与数据库。

## 检查清单

| 项目           | 说明 |
|----------------|------|
| 飞书「即将抢注」 | 通知设置中 Webhook 已填且启用，执行后应收到卡片。 |
| WHOIS          | 执行过程会查一次目标域名，可看 `backend/logs/` 或控制台是否有 WHOIS 相关错误。 |
| 任务状态       | 可注册时任务会变为 `processing`，结果字段会提示「请尽快手动完成注册」。 |
| GoDaddy        | 未配置或未启用时不会调 GoDaddy，只走「手动分支」；要测真实下单需填 key 并勾选自动注册。 |

---

## 常见问题

- **收不到飞书**：检查后台「通知设置」里 Webhook 是否保存成功、是否启用；再确认 `notify_settings` 表里 `webhook_url`、`enabled=1`。
- **WHOIS 报错**：可能是限频或网络，稍后重试；或换一个域名测试。
- **任务一直是 pending**：说明 WHOIS 返回仍不可注册（未到释放时间），属正常，等域名到可注册后再测或换一个已可注册的测试域名。
- **402 INVALID_PAYMENT_INFO**：GoDaddy 无法扣款。OTE 测试环境需在 [GoDaddy 开发者中心](https://developer.godaddy.com/) 为测试账号绑定支付方式，或联系 GoDaddy 支持为 OTE 开通预充值；生产环境需账户余额或有效支付方式。
